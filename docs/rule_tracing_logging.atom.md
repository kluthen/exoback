# Global Request Traceability Logging

## INTENT
To ensure every distinct system interaction can be traced uniformly across all backend services (Laravel and Go) for debugging, performance monitoring, and auditing.

## THE RULE / LOGIC
**Log Format Constraint:**
At the point of ingress (when an HTTP request reaches Laravel or Go, or an internal event fires), the application MUST record a plain-text log entry. The log structure is rigidly:
```text
[{YYYY-MM-DDTHH:MM:SSZ}] [{ref_id}] [{ENDPOINT_OR_HANDLER}] {Message}
```
*   **Time:** Must be UTC standard timestamp format (e.g., `2026-03-05T08:55:00Z`).
*   **Ref ID:** Must be the first 8 characters of the `request_id` (UUIDv7) strictly defined in [[api_request_id]]. 
    *   *Example:* For `request_id` `018f5a1b-e549-74d3-b1d3-...`, `ref_id` is `018f5a1b`.
*   **Endpoint/Handler:** The route path or the internal dispatcher name (e.g., `/api/v1/auth/login` or `Engine.Tick`).
*   **Message:** The core intent or status.

**Traceability Propagation:**
*   Any subsequent logs generated within the app during the life cycle of that specific request *must* embed the same 8-character `ref_id`.
*   When a request jumps from Laravel -> Go, Laravel must pack the original `request_id` into the proxy payload or headers. Go must ingest it and log its own steps using the *same* 8-character `ref_id`.
*   When Go replies inline to Laravel, the transaction holds the same ID.

**Notification Disconnects:**
*   Asynchronous Webhooks/Events emitted spontaneously by Go (e.g. Turn Clock Expiry, end of computations causing a broadcast) are *not* continuations of a previous request. Go must generate a *new* UUIDv7 `request_id` (as defined in [[api_request_id]]), use its 8-character prefix for Go's logs, and pass that `request_id` in the webhook to Laravel. Laravel will log its receipt and WebSocket broadcast using that *same* new `ref_id` generated by Go.

## EXPECTATION (For Testing)
